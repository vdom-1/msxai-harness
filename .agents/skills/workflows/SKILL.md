# Workflows
All workflows defined in this skill file must be executed autonomously, following every step in sequence without interruption unless an error is encountered.

## Deploy project
CRITICAL: The build system directly outputs normalized home-relative paths starting with ~ (e.g., ~/MSX/...); copy this path exactly as printed without expanding or modifying it. You must pass this path to openmsx wrapped inside literal square brackets as [file normalize "~/your/path/here"], because the brackets are actual TCL command execution syntax, not documentation placeholders.

Progress:
- [ ] Step 1: **openmsx_control_sendCommand** `set power off`
- [ ] Step 2: **openmsx_control_sendCommand** `carta eject`
- [ ] Step 3: **openmsx_control_sendCommand** `debug symbols files` (List existing symbols files)
- [ ] Step 4: **openmsx_control_sendCommand** `debug symbols remove <filename>` (Remove each file path found in the Step 3 list)
- [ ] Step 5: **Run in terminal** `make -f build.mk PROJECT=<project_name> TYPE=<object_type>`
- [ ] Step 6: **openmsx_control_sendCommand** `debug symbols load [file normalize "<filename>"]` (Load symbols file generated by Step 5)
- [ ] Step 7: **openmsx_control_sendCommand** `carta insert [file normalize "<filename>"]` (Insert object file generated by Step 5)
- [ ] Step 8: **openmsx_control_sendCommand** `set throttle off`  
- [ ] Step 9: **openmsx_control_sendCommand** `set power on`
- [ ] Step 10: **openmsx_control_sendCommand** `after 500 set throttle on` (**REQUIRED**)
